Conclusion:
Global-local v1 is safe in log-only mode and useful for risk-aware analysis.
However, it is too conservative and has internal consistency issues.
It should not be used for apply.
Next version should enforce:
- if trajectory_risk is high, should_change_original must be false.
- if selected candidate fails deterministic metrics, should_change_original must be false.
- if no candidate is clearly safe, selected_candidate_id should be the original planner choice.
- reason should not be used as a decision signal.
